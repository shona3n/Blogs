doxygen的配置使用

进入项目目录：

生成Doxyfile文件，如果不想文件中有太多注释语句，使用：

修改配置文件Doxyfile。打开这个Doxygen文档，对其中某些字段做配置，一般来说，只需要配置其中十几个字段就可以：

  • 项目名称，将作为于所生成的程序文档首页标题

  • 文档版本号，可对应于项目版本号，譬如 svn 、 cvs 所生成的项目版本号

  • 程序文档输出目录

  • 程序文档语言环境

  • 指定需要生成的文件的路径，或者指定文件

  • 如果是制作C程序文档，该选项必须设为YES，否则默认生成C++文档格式

  • 对于使用 typedef 定义的结构体、枚举、联合等数据类型，只按照 typedef 定义的类型名进行文档化

  • 在C++程序文档中，该值可以设置为NO，而在C程序文档中，由于C语言没有所谓的域/名字空间这样的概念，所以此处设置为YES

  • 让doxygen静悄悄地为你生成文档，只有出现警告或错误时，才在终端输出提示信息

  • 只对头文件中的文档化信息生成程序文档

  • 递归遍历当前目录的子目录，寻找被文档化的程序源文件

  • 示例程序目录

  • 示例程序的头文档 (.h 文件 ) 与实现文档 (.c 文件 ) 都作为程序文档化对象

  • 递归遍历示例程序目录的子目录，寻找被文档化的程序源文件

  • 允许程序文档中显示本文档化的函数相互调用关系

  • 不生成 latex 格式的程序文档

  • 在程序文档中允许以图例形式显示函数调用关系，前提是你已经安装了graphviz软件包

  • 让 doxygen 从配置文件所在的文件夹开始，递归地搜索所有的子目录及源文件

  • 在最后生成的文档中，把所有的源代码包含在其中

  • 这会在 HTML 文档中，添加一个侧边栏，并以树状结构显示包、类、接口等的关系
